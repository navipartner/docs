# Outdated Documentation Report: NP Pay & eCommerce

**Date:** 2026-03-14  
**Scope:** NP Pay (20 articles) and eCommerce (79 articles) documentation  
**Method:** Cross-referenced docs against navipartner/npcore-partners codebase on GitHub

---

## Executive Summary

Analysis of the NP Pay and eCommerce documentation against the current codebase reveals **4 Critical**, **8 High**, and **7 Medium** priority issues. The most impactful findings are:

1. The **Raptor integration module** is marked obsolete in code (Jan 2026) but documented as active
2. The **Pay by Link Setup** table is obsolete — fields moved to NP Pay Setup (Adyen Setup)
3. The **Shopify Integration Setup** page has had most fields removed and moved to store-specific settings
4. The **NP Pay overview** incorrectly states only cloud architecture is supported — local integration now exists

---

## CRITICAL Priority Findings

### C1. Raptor Integration Module is Obsolete
**Files affected:**
- `content/en/docs/ecommerce/raptor/setup/index.md`
- `content/en/docs/ecommerce/raptor/create_actions/index.md`
- `content/en/docs/ecommerce/raptor/_index.md`

**Evidence:** In `npcore-partners`, all Raptor code files reside in `Application/src/__Legacy (STILL SUPPORTED)/Raptor/` and carry:
```
ObsoleteState = Pending;
ObsoleteTag = '2026-01-29';
ObsoleteReason = 'This module is no longer being maintained';
```

This applies to `RaptorSetup.Table.al`, `RaptorSendData.Codeunit.al`, `RaptorAction.Table.al`, `RaptorManagement.Codeunit.al`, `RaptorAPI.Codeunit.al`, and all related files.

**Specific issues:**
- Docs reference `Codeunit 6151494 "Raptor Send Data"` — actual name is `"NPR Raptor Send Data"` (NPR prefix)
- Entire setup and action creation documentation describes a module that is being deprecated
- No deprecation notice exists in any of the Raptor documentation
- These docs present Raptor as a fully functional, current integration

**Recommendation:** Add deprecation banners to all Raptor docs, or move them to a legacy/archived section.

- **Loyalty Points as Metafield** (boolean)
- **Update Cust. Phone No. from BC** (boolean)
- **BC Customer Transactions** (boolean) — POS transaction export
- **Auto Set as Shopify Item** (boolean)
- **Auto Update Items from Shopify** (boolean)
- **Auto Update Cust. from Shopify** (boolean)
- **Send Order Ready for Pickup** (boolean) — C&C order support
- Various webhook existence indicators
- **Sales Price on Order Lines** (enum)
- **Invent.Level Update Batch Size** (integer)
- **Get Payment Lines from Shopify** now has 3 options: "Before Capture, On Order Import, **Both on Import and before Capture**" (docs show only 2)

**Recommendation:** Update the Shopify Store reference doc with all new fields and their descriptions.
